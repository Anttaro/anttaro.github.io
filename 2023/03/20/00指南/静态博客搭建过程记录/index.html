2.2 NexT主题美化

NexT默认主题略有些简陋，希望DIY以下功能：加入左栏的标签、归档等功能；单篇文章可以在左栏显示大纲目录（后来发现大纲目录是默认自带的功能）。注：本地编辑.md文件可以打出空行，但是在静态博客上空行不会被显示，因此排版时尽量使用分隔线代替空行。

2.3 标签、分类和目录

参考文章见：NexT主题使用攻略基础——添加分类、标签及关于。

2.3.1 标签

标签页生成：在hexo文件夹中（沿用第一篇参考文章的设定）右键Git Bash Here，输入hexo new page tags，运行后如图：

在上面显示的路径：hexo\source\tags 文件夹里找到 index.md 文件，打开这个文件，在 front matter 中、date那一行下面，紧接着加上一行type: "tags"。写文章要加入标签时，在 front matter 部分加入tags: [标签1,标签2]就可以了。

配适手机端，把这个 layout 选项改为 true：

2.3.2 分类

类似地，在 hexo 文件夹中右键 Git Bash Here，输入 hexo new page categories，运行后按照输出找到这个文件：hexo\source\categories\index.md，打开该文件，在 front matter中、date 那一行之后，添加代码 type: "categories"。需要将分类赋予文章的时候与标签同理，在文章的 front matter 中添加。

2.3.3 目录

在 next 主题的配置文件中更改。目录打开自动换行，关闭目录自动编号，关闭多级目录全部展开

注：在首页中，希望不要显示完整文章，而是只显示一部分文字预览，可以在编辑文章的时候加入<!-- more -->来手动隔断显示和隐藏的部分。

2.4 一些tips

在hexo/source/_posts文件夹中创建新文件的时候，可以在_post文件夹中任意创建子文件夹，以便于管理文档，但是在部署之后，引用这些文档的地址会根据子文件夹的不同而变化，导致编写目录、访问链接上的麻烦。解决方式是修改配置文件，保证文章链接一经生成永不更改，这个功能目前还未实现，参考文章：使用子文件夹管理 Hexo 文章且不改变文章永久链接

在每个文件最开头，标明文章标题、发布时间等的那个“题头”叫做“Front Matter”，需要查询其相关功能的时候可以搜关键词。更多关于 typora 的使用技巧参看这篇文章：Typora 完全使用详解

3. 购买自己的域名（非必须）

在不挂梯子的情况下，github.io网页访问比较不稳定，有时候可以进，有时候加载不出来。自己买一个域名可以更方便地访问。2023-03-23更新：目前看来访问还是比较稳定的，暂时就不用自己DIY域名了。

4. 未雨绸缪 - 如果未来要换电脑

hexo\source文件夹里面存储了所有.md文件，包括文章、标签和图片。hexo的配置文件与NexT的配置文件需要编辑。或许直接把hexo文件夹移过去？
